# Complete Guide to n8n-mcp: AI-Powered Workflow Automation with Claude Desktop

n8n-mcp is a revolutionary Model Context Protocol (MCP) server that brings AI-powered workflow automation to your fingertips. With support for 525+ n8n nodes and 99% property coverage, it enables you to create, manage, and execute complex workflows using natural language through Claude Desktop.

## What is n8n-mcp?

n8n-mcp bridges the gap between AI assistants and n8n workflow automation platform. It allows you to:

- Create workflows using natural language descriptions
- Manage existing workflows through AI conversations
- Execute and monitor workflow runs
- Validate workflow configurations automatically
- Reduce configuration errors with smart node discovery

## Prerequisites

Before getting started, ensure you have:

- Node.js installed on your system
- Claude Desktop application
- n8n installed locally (see installation steps below)

## Installing n8n Locally

First, you'll need to install n8n on your local machine. n8n is a self-hostable workflow automation platform with 400+ integrations.

### Option 1: Using npx (Quickest Method)

The fastest way to get n8n running locally:

```bash
npx n8n
```

This command will:
- Download and install n8n automatically
- Launch the n8n interface
- Make it available at `http://localhost:5678`

### Option 2: Using Docker (Recommended for Production)

For a containerized setup with persistent data:

```bash
# Create a data volume for persistence
docker volume create n8n_data

# Run n8n container
docker run -it --rm \
  --name n8n \
  -p 5678:5678 \
  -v n8n_data:/home/node/.n8n \
  docker.n8n.io/n8nio/n8n
```

### Accessing Your Local n8n Instance

Once started, n8n will be available at:
- **URL**: `http://localhost:5678`
- **Default Port**: 5678

### Initial n8n Setup

On first startup:
1. Open `http://localhost:5678` in your browser
2. Create your admin user account
3. Set up your workspace preferences
4. Explore the 400+ available integrations

### Getting Your n8n API Key

For n8n-mcp integration, you'll need an API key:

1. In your n8n interface, go to **Settings**
2. Navigate to **API Keys**
3. Click **Create API Key**
4. Give it a descriptive name (e.g., "n8n-mcp-integration")
5. Copy the generated key for later use

## Claude Desktop Configuration

### Basic Configuration (Documentation Tools Only)

For basic functionality that includes documentation and workflow creation tools, add this configuration to your Claude Desktop settings:

```json
{
  "mcpServers": {
    "n8n-mcp": {
      "command": "npx",
      "args": ["n8n-mcp"],
      "env": {
        "MCP_MODE": "stdio",
        "LOG_LEVEL": "error",
        "DISABLE_CONSOLE_OUTPUT": "true"
      }
    }
  }
}
```

### Advanced Configuration (Full n8n Management)

For complete n8n instance management including workflow deployment and execution monitoring, use this configuration:

```json
{
  "mcpServers": {
    "n8n-mcp": {
      "command": "npx",
      "args": ["n8n-mcp"],
      "env": {
        "MCP_MODE": "stdio",
        "LOG_LEVEL": "error",
        "DISABLE_CONSOLE_OUTPUT": "true",
        "N8N_API_URL": "http://localhost:5678/api/v1",
        "N8N_API_KEY": "your-api-key"
      }
    }
  }
}
```

**Important Note**: For local n8n instances, use `http://localhost:5678/api/v1` as the N8N_API_URL. Note that it's **HTTP** (not HTTPS) for local development, and includes the `/api/v1` path suffix.

### Configuration Steps

1. Open Claude Desktop
2. Go to Settings â†’ MCP Servers
3. Add the configuration JSON above
4. Replace `your-n8n-instance.com` with your actual n8n instance URL
5. Replace `your-api-key` with your n8n API key
6. Restart Claude Desktop

## Testing Workflows with Postman

### Setting Up Postman Tests

1. **Export Workflow from Claude**
   ```
   Export the customer data workflow as JSON
   ```

2. **Import into n8n**
   - Copy the JSON workflow from Claude's response
   - Open your n8n instance
   - Click "Import from JSON"
   - Paste the workflow JSON

3. **Create Postman Collection**
   ```json
   {
     "info": {
       "name": "n8n Workflow Tests",
       "description": "Testing n8n workflows created with Claude"
     },
     "item": [
       {
         "name": "Trigger Webhook Workflow",
         "request": {
           "method": "POST",
           "header": [
             {
               "key": "Content-Type",
               "value": "application/json"
             }
           ],
           "body": {
             "mode": "raw",
             "raw": "{\n  \"customer_id\": \"12345\",\n  \"email\": \"test@example.com\",\n  \"action\": \"signup\"\n}"
           },
           "url": {
             "raw": "https://your-n8n-instance.com/webhook/test-workflow",
             "protocol": "https",
             "host": ["your-n8n-instance", "com"],
             "path": ["webhook", "test-workflow"]
           }
         }
       }
     ]
   }
   ```

### Testing Workflow Execution

1. **Test Webhook Triggers**
   ```bash
   curl -X POST https://your-n8n-instance.com/webhook/test-workflow \
   -H "Content-Type: application/json" \
   -d '{"test": "data"}'
   ```

2. **Verify Workflow Runs**
   - Check n8n execution history
   - Monitor logs through Claude: "Show me the last execution of the test workflow"

3. **Validate Output Data**
   - Test endpoint responses
   - Verify data transformations
   - Check error handling
